# Caret — Design Context

## Design Context

### Users

**Who they are:**
Caret serves a diverse audience united by a common need: intelligent, distraction-free document creation.

- **Students under pressure**: Working against academic deadlines, need clarity, guidance, and stress reduction
- **Focused professionals**: Crafting proposals, reports, and documentation requiring confidence and efficiency
- **Creative writers in flow**: Authors and bloggers seeking a distraction-free creative space with minimal friction
- **Researchers organizing**: Academic researchers structuring complex ideas, needing powerful organization

**Their context:**
These users span from high-stress deadline situations to contemplative deep work. They switch between modes: outlining structure, drafting content, refining language, and collaborating with peers. AI assistance should feel like a capable partner, not a separate chat window.

**The job to be done:**
Transform ideas into polished, structured documents efficiently while maintaining creative control. Unlike code (where AI agents have thrived in tools like Cursor and Copilot), document writing has remained stagnant with superficial AI integrations (Word Copilot, Google Docs Gemini). Caret bridges this gap with true agentic capabilities woven into the editing experience.

### Brand Personality

**Three words:**
Confident. Minimal. Focused.

**Voice & Tone:**
- **Confident without arrogance**: The tool empowers users, making them feel in command with AI as a capable assistant
- **Clear without sterility**: Swiss design rigor meets approachable warmth — minimal but never cold
- **Intelligent without showing off**: AI capabilities are integrated naturally, not showcased as a separate feature

**Emotional goal:**
Caret should evoke **confidence and control**. Users should feel empowered to create their best work, with AI enhancing their capabilities rather than replacing their judgment. The interface fades into the background, letting content take center stage.

### Aesthetic Direction

**Visual tone:**
"Swiss Focus" — a rigorous, grid-based, content-first aesthetic that mimics high-end digital paper.

**Core principle:**
Great writing deserves a great environment. The interface should feel like the digital equivalent of a premium notebook: minimal chrome, impeccable typography, and thoughtful spatial rhythm.

**Color strategy:**
- **Primary palette**: TWO colors only — Blue (tool/UI) and Orange (user focus/caret/AI)
  - **Blue (#0066CC light / #3B99FC dark)**: Primary UI color reminiscent of trusted document tools (Word, Docs) but refined
  - **Orange (#FF4500 light / #FF6B35 dark)**: The signature brand color representing user focus. Originally just the caret, now extended to AI features — AI is not separate, it's the new normal
- **Neutrals**: Off-white (#FAFAFA) and true dark (#121212) backgrounds with subtle grays for hierarchy
- **Rationale for simplification**: Purple was removed to maintain visual simplicity and avoid treating AI as a separate entity. AI is integrated into the core experience, not a special feature.

**Typography:**
- **Document canvas**: Merriweather serif (18px, 1.7 line height) for elegant readability
- **UI chrome**: Inter sans-serif for clean, utilitarian interface elements
- **Swiss heading style**: No bold headings — hierarchy through size and spacing only (400 weight, -0.02em tracking)
- **Accessibility**: Font choices and spacing optimized for dyslexia/reading support with clear letter-spacing and generous line height

**Spacing & rhythm:**
- 4px base unit for mathematical consistency
- Generous whitespace to reduce cognitive load
- Grid-based layout for visual order

**References (what to emulate):**
- **Arc browser**: Bold, reimagined aesthetic that feels fresh without sacrificing usability
- **lex.page**: Minimalist AI writing tool with clean typography and subtle AI integration (closest conceptual match)
- **Linear**: Sophisticated, polished, keyboard-first interactions with refined animations (though slightly darker than Caret's target)
- **Notion**: Flexible, approachable for diverse users without feeling cluttered
- **Craft / Bear**: Beautiful native writing apps with typography-first design

**Anti-references (what to avoid):**
- **Microsoft Office**: Cluttered ribbons, overwhelming toolbars, dated enterprise feel
- **Generic AI chatbots**: AI should be woven into editing, not a separate chat panel bolted on
- **Minimal to a fault**: Not sterile or intimidating — warmth through color, motion, and thoughtful details

### Design Principles

These principles guide every design and development decision in Caret:

1. **Content is sovereign**
   - The document surface is sacred. UI chrome fades to 20% opacity during focus mode (triggered after 2s inactivity)
   - Generous reading width (800px, 900px on wide displays), optimal line length (60-80 characters)
   - Typography hierarchy through size and spacing, never through bold or color (Swiss typography rules)

2. **AI is integrated, not adjacent**
   - No purple "AI features" branding — AI uses the same orange as user actions because it's the new normal
   - Inline ghost text suggestions, streaming edits, and contextual enhancements feel native to the editing experience
   - AI never writes directly to the database; all changes flow through Tiptap Transactions to maintain Y.js sync

3. **Confidence through clarity**
   - Every UI element has a clear purpose and visual hierarchy
   - Error messages are actionable, not technical
   - Keyboard shortcuts and interactions are consistent and discoverable
   - Users should always know what the system is doing and feel in control

4. **Minimal with warmth**
   - Swiss rigor meets human touch: grid-based layouts with subtle animations (landing page only via Framer Motion)
   - The orange caret is the brand's heartbeat — representing the user's active presence
   - Thoughtful microinteractions that feel alive without being distracting
   - Dark mode is first-class (not an afterthought) with carefully tuned shadow opacity

5. **Accessibility is non-negotiable**
   - WCAG AA minimum (4.5:1 contrast ratios)
   - Font choices optimized for dyslexia/reading support (Merriweather's generous spacing, Inter's clear letterforms)
   - All decorative/continuous animations gated behind `useReducedMotion()`
   - Keyboard navigation and screen reader support for all critical workflows
   - Focus indicators visible (3px ring, accent-main at 40% opacity)

## Technical Design Specifications

### Color Palette

**Light Mode:**
| Token | Hex | Usage |
|:---|:---|:---|
| `bg-app` | #FAFAFA | Global background (off-white reduces eye strain) |
| `bg-surface` | #FFFFFF | Document "sheet" |
| `text-primary` | #1A1A1A | Main content |
| `text-secondary` | #6E6E73 | UI labels, metadata |
| `accent-main` | #0066CC | Primary UI color (buttons, links) |
| `accent-caret` | #FF4500 | **User focus + AI features** (signature brand color) |
| `border-subtle` | #E5E5E5 | Panel separators |

**Dark Mode:**
| Token | Hex | Usage |
|:---|:---|:---|
| `bg-app` | #121212 | True dark background |
| `bg-surface` | #1E1E1E | Document surface (elevated) |
| `text-primary` | #E8E8E8 | High contrast text |
| `text-secondary` | #9CA3AF | UI labels |
| `accent-main` | #3B99FC | Brighter blue for visibility |
| `accent-caret` | #FF6B35 | Softer orange for dark backgrounds |
| `border-subtle` | #2A2A2A | Subtle borders |

### Typography Scale

| Element | Font | Size | Line Height | Weight | Tracking |
|:---|:---|:---|:---|:---|:---|
| **Display (H1)** | Merriweather | 32px | 1.3 | 400 | -0.02em |
| **Heading (H2)** | Merriweather | 28px | 1.3 | 400 | -0.02em |
| **Subheading (H3)** | Merriweather | 24px | 1.4 | 400 | -0.02em |
| **Body Text** | Merriweather | 18px | 1.7 | 400 | normal |
| **UI Large** | Inter | 16px | 1.5 | 500 | normal |
| **UI Base** | Inter | 14px | 1.5 | 400 | normal |
| **UI Small** | Inter | 12px | 1.4 | 400 | 0.02em |
| **Code** | Fira Code | 14px | 1.6 | 400 | normal |

### Layout Grid

- **Base unit**: 4px (all spacing follows this scale)
- **Document max-width**: 800px (900px on >1440px displays)
- **Top bar height**: 56px (fixed)
- **Gutter**: 24px (between major UI sections)

### Z-Index Layers

| Layer | Value | Usage |
|:---|:---|:---|
| **z-0** | 0 | Document surface (base canvas) |
| **z-10** | 10 | Decorators (highlights, AI diffs) |
| **z-20** | 20 | Collaboration (carets, cursors) |
| **z-30** | 30 | Chrome (toolbars, navigation) |
| **z-40** | 40 | Floating UI (panels, dropdowns) |
| **z-50** | 50 | Modals, overlays |
| **z-100** | 100 | Tooltips, toasts |

### Border Radius (Swiss Scale)

- `radius-none`: 0px (document canvas)
- `radius-sm`: 2px (checkboxes, tooltips)
- `radius-base`: 4px (buttons, inputs — standard)
- `radius-md`: 6px (cards, modals)
- `radius-lg`: 8px (large containers)
- `radius-full`: 9999px (avatars only)

### Animation Standards

**Timing:**
- Hover/focus: 150ms ease-out
- Panel open/close: 250ms ease-out
- Page transitions: 300ms ease-in-out
- Toast enter/exit: 200ms ease-out

**Motion principles (Framer Motion — landing page only):**
- Animate only `transform` and `opacity` (GPU-accelerated)
- Gate all decorative/continuous animations behind `useReducedMotion()`
- Spring config: `stiffness: 110–260 / damping: 18–24`
- Never exceed 5° tilt on 3D effects
- Parallax scroll max: 60px y-offset, 0.7 min opacity

### Key Behaviors

**Focus Mode:**
- Trigger: 2 seconds after last cursor/scroll activity
- Effect: Peripheral UI fades to 20% opacity over 200ms
- Remains: Document surface, orange caret, essential editing tools
- Return: Hover on any peripheral UI element restores full opacity

**Theme Toggle:**
- Default: Respect system preference (`prefers-color-scheme`)
- Manual override: Persist in `localStorage` as `caret-theme`
- Smooth transitions: 200ms ease for background, color, border

**AI Integration:**
- Streaming text: Character-by-character reveal (30ms per char)
- Ghost text suggestions: `color: text-ghost`, `pointer-events: none`
- All AI actions use `accent-caret` (orange) — AI is not separate, it's integrated

## Component Architecture

**Feature-first structure:**
```
src/
├── features/
│   ├── editor/          # Tiptap implementation
│   ├── ai-assistant/    # AI chat panel, streaming
│   ├── collaboration/   # Y.js multiplayer
│   ├── auth/           # Authentication
│   ├── landing/        # Public landing page
│   └── settings/       # App settings
├── components/
│   ├── ui/             # Primitives (Button, Input, Logo)
│   └── layout/         # TopBar, MainLayout
├── hooks/              # use_theme, use_focus_mode
├── stores/             # Zustand (auth, theme, tabs, ai)
└── locales/            # i18n (en-US, es, fr, de, pt)
```

**State management:**
- **Editor state**: Tiptap (ProseMirror) — accessed only via `useEditor` hook, never duplicated in React state
- **Global UI**: Zustand (theme, sidebar visibility, user session)
- **Server state**: TanStack Query (documents, folders with caching)
- **Form/local**: React `useState` (ephemeral, component-scoped)

## Accessibility Standards

**WCAG AA Compliance:**
- Contrast ratios: 4.5:1 for normal text, 3:1 for large text
- Focus indicators: 3px ring with `accent-main` at 40% opacity
- Keyboard shortcuts: All critical actions accessible without mouse
- Screen reader: `aria-live="polite"` on AI streaming regions
- Motion: All decorative animations gated behind `useReducedMotion()`

**Dyslexia/Reading Support:**
- Font choice: Merriweather's generous letter-spacing and distinct letterforms
- Line height: 1.7 for body text (above standard 1.5 recommendation)
- Spacing: Clear visual rhythm with 4px base unit prevents crowding
- Contrast: High contrast ratios exceed minimum requirements
